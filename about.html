WRITING DIRECTION

内容方向

  • 像素世界构建：地图、地形、天气、叙事空间。
  • 创意编码：shader、调色板、渲染策略、交互细节。
  • 个人项目日志：设计选择、踩坑记录、版本演进。
  • 声音实验：chiptune、Web Audio、小型声景设计。
DESIGN RULES

博客维护原则

  • 氛围要强，但不能压过正文。
  • 像素字体只做强调，长文本保持可读。
  • 公开导航里的链接必须真实可达。
  • 能静态部署的功能，不引入复杂后端。
WHY THIS VERSION

这个博客保持像素氛围，但结构以长期更新为目标。

  • 文章是独立页面，不再依赖浮层交互。
  • 像素风格集中在头图和场景装饰，正文排版专注可读性。
  • 目录结构适合 GitHub Pages，以后也方便切到静态站生成器。
  • 公开导航区分内容 / 项目 / 模板 / 关于四种入口。
CONTENT SYSTEM

每一页只承担一个职责。

首页负责导览与精选，归档页负责文章检索，项目页负责作品记录，模板页负责设计支撑，关于页负责联系入口。这样长期更新时不会到处重复维护同一件事。
